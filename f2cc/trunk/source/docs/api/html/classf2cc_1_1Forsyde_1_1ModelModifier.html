Detailed Description

The ModelModifier is a class which provides a set of model modification methods. The modifications are such that they preserve the semantics of the model, and are used to simplify the latter synthesis process or affect the structure of the generated code (i.e. whether to generate sequential C code or parallel CUDA C code).

std::list<ContainedSection> f2cc::Forsyde::ModelModifier::findContainedSections ( ) throw (IOException, RuntimeException) [private]

Searches for contained sections within the model. A contained section is a part of the model which:

  • start with a unzipxSY process, and
  • ends with a zipxSY process, where
  • all data flow diverging from the starting point converges at the end point, and
  • all data flow converging to the end point diverges from the starting point. This method cannot handle nested levels of containment and thus only finds the closest layer of containment.
Returns:
List of contained sections.

Checks if a contained section is data parallel. A data parallel section is a section:

  • where there is at least one process between the start and the end point,
  • where all its diverging paths are of equal lengths,
  • which consist only of mapSY processes, and
  • where all processes at the same distance from the starting point are pairwise equal (i.e. has the same function argument).

Redirects data flow between two processes A and B to another two processes C and D. The data flow is redirected by adding the in ports of A to C, and the out ports of B to D. This will break all connections to A and from B, thus unconnecting that segment from the model (but it does not delete it). Processes C and D can be the same process.
