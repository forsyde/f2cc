#define THROW_EXCEPTION (   exception_class,

)    throw exception_class(__FILE__, __LINE__, ##__VA_ARGS__)

Throws an exception.

All exceptions require that the name of the source file and the line from which the exception is thrown. Although there are preprocessor macros for getting the file name and line (__FILE__ and __LINE__), this would be tedious if they would have to be inserted manually into the constructor arguments of the exception (using the macros inside the constructor doesn't work). By using this macro, these required values are automatically added as arguments to the exception so the programmer needn't bother.

The macro takes an exception class as mandatory argument, along with a variable list of additional arguments. Thus, an exception of type ExampleException, which requires an additional int argument, can be thrown as follows: 

    THROW_EXCEPTION(ExampleException, 10);

This is equivalent to writing: 

    throw ExampleException(__FILE__, __LINE__, 10);

If the exception has no additional arguments, the variable list of arguments must be removed entirely, like so: 

    THROW_EXCEPTION(ExampleException);

The following is an incorrect use of the macro and causes an syntax error when compiled: 

    THROW_EXCEPTION(ExampleException, );
Parameters:
exception_className of the exception class to throw.
...A variable list of additional arguments needed by the exception constructor. If there are no such arguments, remove this parameter entirely in the macro call (see description).
